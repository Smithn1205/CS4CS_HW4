I will write a paragraph for each methodology of Software Development Lifecycle(SDLC), there are many of them but I will suggest a few. My suggestion is as follows:

  1. Capability Maturity Model (CMM): This model helps the company to improve their software. It provides a structed way to improve their software. It has 5 levels initial, repeatable, defined, managed, and optimizing.
  2. Waterfall Model: Here, the levels are arranged in the form a waterfall. Before going to the next level the previous level should be carried out. The phases of the waterfall model are requirement, design, development, testing, deployment and maintainance.
  3. V-Shaped Model: This model is arranged in a V shape with testing done at each stage. The design is as follows: requirement gathering and analysis, design, implementation, testing, deployment and maintainance.
  4. Structured Evolutionary Prototyping: In this model, you create a demo model first and let the users use it first to find out any bugs or vulnerabilities. The main steps are requirement gathering and analysis, quick design, build a prototype, initial user evaluation and refining prototype.
  5. Rapid Application Development (RAD): In this type of model, the team creates few small models for users to test and improve them until the final software is ready. It helps to buld software much faster.
  6. Incremental Model:In this model, the team builds the software part by part. When each version is completed, it is merged together. Its phases are requirement analysis, design and development, deployment and testing and implementation.
  7. Spiral Model: It is one of the most important SDLC model. It follows systematic and iterative approach. It looks like a spiral with many loops. Its phases include objective defined, risk analysis, engineering, evaluation and planning.
  8. Agile Model: It works in small pieces, called as sprints, to deliver and adjust the parts quickly to develop the final product. Its methods are Dynamic Systems Development Method (DSDM), Feature-driven development (FDD), Extreme Programming (XP), Scrum, etc.

Among so many SDLC methodologies the ones that are in the AGILE classification are better. Among AGILE SDLC methods, I also have a few suggestions:

  1. Scrum: It works on a simple idea, the team works on small parts of the projet and combine them after each part is completed. It is simple, effective and manageable.
  2. Dynamic Systems Development Method (DSDM): It focuses on delivering the project on time, keeping in mid the budget.
  3. Feature-driven development (FDD): This method is structured and focuses on delivering the working features in a simple manner.
  4. Extreme Programming (XP): This model focuses on customer satisfaction and frequently keeps on releasing the updates.
  5. Kanban: It is easy to implement. It helps the team to manage there work efficiently.

I have used CSS with the webpage and there are 3 ways to include CSS in a webpage:

  1. External CSS: External CSS is when a user creates a seperate page to design the webpage using CSS. The page name should end with ".css".
  2. Internal CSS: Internal CSS is when a user writes the CSS styles within the head tag, between a style tag.
  3. Inline CSS:Inline CSS is when the user writes the CSS styles infront of each line using style attribute.

The reason CSS is used in a webpage are:

  1. CSS is used to enhance the webpage.
  2. The webpage doesn't require any extra time to load. So, it is effective.
  3. It helps to make the webpage more interactive so the users can easily navigate through the webpages.
